# CS-350 Portfolio Reflection

## Project Summary

For my portfolio, I selected the Thermostat Final Project and the Smart Thermostat State Machine because they best represent what I learned throughout this course. The project involved building a smart thermostat using a Raspberry Pi, Python, sensors, buttons, LEDs, an LCD display, and communication protocols such as I2C and UART. The thermostat monitors the current temperature, allows the user to change the desired temperature, and switches between OFF, HEAT, and COOL operating modes.

## What I Did Well

One of my biggest strengths was troubleshooting hardware and software together. Throughout the project, I tested each component individually, verified GPIO assignments, and worked through wiring problems until everything functioned correctly. I also successfully integrated multiple hardware peripherals into one complete embedded system.

## Where I Could Improve

If I completed this project again, I would spend more time planning the hardware layout before wiring everything together. This would reduce troubleshooting time and make debugging easier. I would also improve my documentation while developing the project instead of waiting until the end.

## Tools and Resources

This project showed me the value of using official documentation, GPIO pin diagrams, debugging tools, and testing hardware one component at a time. These resources will continue to be useful in future embedded systems and programming projects.

## Transferable Skills

The skills I developed during this project will transfer directly to future coursework and professional work. I gained experience with Python programming, debugging, hardware integration, state machines, GPIO, I2C, UART communication, and systematic problem solving. I also learned that embedded systems require both software and hardware knowledge to successfully identify and solve problems.

## Maintainability, Readability, and Adaptability

I organized my code into separate functions for each task, used descriptive variable names, and kept related functionality together. This made the program easier to read, test, and modify as new features were added throughout the course. Building the project one milestone at a time also made it easier to expand into the final thermostat application.
